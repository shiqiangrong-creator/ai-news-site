## 三、技术方案对比

### 方案一：静态网站 + 定时抓取（推荐）

**技术栈**：Hugo/Next.js + GitHub Actions + RSS解析

```
┌─────────────────────────────────────────────────────────┐
│  GitHub Actions (定时任务)                               │
│  ├── 每日自动执行爬虫脚本                               │
│  ├── 解析RSS源 (feedparser)                            │
│  ├── 生成静态HTML/JSON                                 │
│  └── 自动提交到GitHub Pages                            │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  静态网站 (Vercel/GitHub Pages)                         │
│  ├── 首页：文章列表                                     │
│  ├── 分类页：大模型、AI应用、技术解读                   │
│  └── 标签页：按关键词分类                               │
└─────────────────────────────────────────────────────────┘
```

**优势**：
- ✅ 完全免费（GitHub Pages / Vercel Hobby）
- ✅ 零服务器维护
- ✅ 加载速度快（CDN加速）
- ✅ 安全性高（无数据库、无后端漏洞）
- ✅ 高可用性（Vercel提供99.9% SLA）

**劣势**：
- ❌ 搜索功能需要借助第三方（如Algolia）
- ❌ 内容更新依赖定时任务（非实时）
- ❌ 扩展性有限（如需用户系统则不适用）

**成本估算**：
| 项目 | 费用 |
|------|------|
| GitHub Pages | 免费 |
| Vercel Hobby | 免费（100GB带宽/月） |
| 域名 | ~¥50/年（可选） |
| **总计** | **免费** |

## 六、代码示例

### RSS解析脚本（Python）

```python
import feedparser
import json
from datetime import datetime

def fetch_ai_news():
    sources = {
        "量子位": "https://www.qbitai.com/feed",
        "机器之心": "https://rsshub.app/weixin/gzh/almosthuman2014"
    }
    
    articles = []
    for name, url in sources.items():
        feed = feedparser.parse(url)
        for entry in feed.entries[:10]:
            articles.append({
                "title": entry.title,
                "link": entry.link,
                "source": name,
                "date": datetime.now().isoformat()
            })
    
    # 输出JSON供Hugo使用
    with open("data/news.json", "w", encoding="utf-8") as f:
        json.dump(articles, f, ensure_ascii=False, indent=2)

if __name__ == "__main__":
    fetch_ai_news()
```

### GitHub Actions工作流

```yaml
name: Daily AI News Update
on:
  schedule:
    - cron: '0 0 * * *'  # 每天UTC 0点（北京时间8点）
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install feedparser
      - run: python scripts/fetch_news.py
      - run: hugo
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
```

## 七、需要用户确认的关键决策点

### 🔴 决策点 1：网站定位

| 选项 | 说明 |
|------|------|
| **A. 纯个人使用** | 简洁文章列表，无需复杂功能 |
| **B. 对外分享** | 需要更好UI、访问统计、SEO优化 |

**建议**：先从A开始，快速验证后再扩展

---

### 🔴 决策点 2：功能范围

| 功能 | 优先级 | 备注 |
|------|--------|------|
| 文章列表展示 | 必须 | 基础功能 |
| 分类筛选 | 推荐 | 按来源/领域分类 |
| 关键词搜索 | 建议 | 可后加 |
| 收藏功能 | 可选 | 需要用户系统 |
| AI摘要 | 高级 | 需要API成本 |

---

### 🔴 决策点 3：展示内容

| 选项 | 说明 |
|------|------|
| **A. 仅标题+链接** | 简洁，数据量小 |
| **B. 标题+摘要+链接** | 平衡方案，展示预览 |
| **C. 全文抓取** | 内容丰富，但版权风险高 |

**建议**：选择B，仅存储摘要，原文链接跳转原文

---

### 🔴 决策点 4：更新频率

| 选项 | 频率 | 说明 |
|------|------|------|
| **A. 每日更新** | 1次/天 | 适合晨间阅读 |
| **B. 每6小时** | 4次/天 | 实时性较好 |
| **C. 每小时** | 实时 | 资源消耗大 |

**建议**：选择A每日更新，配合GitHub Actions免费额度

---

### 🔴 决策点 5：是否需要AI摘要

| 选项 | 说明 |
|------|------|
| **否** | 纯链接聚合，0成本 |
| **是** | 调用DeepSeek/OpenAI API，约¥10/月 |

**建议**：初期选择否，后续按需添加

## 八、风险与注意事项

### ⚠️ 法律合规

- 仅抓取公开RSS源数据
- 尊重原文版权，仅展示摘要和链接
- 避免存储完整文章内容
- 必要时添加免责声明

### ⚠️ 技术风险

| 风险 | 应对措施 |
|------|----------|
| RSS源失效 | 多源备份，定期检查 |
| 反爬限制 | 使用RSS而非网页抓取 |
| 免费额度超限 | 监控使用量，及时优化 |
| 第三方服务不稳定 | 优先使用官方RSS |
